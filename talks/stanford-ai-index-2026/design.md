# Japandi Web/UI Design System v2

一句话定义：**克制、温暖、安静、可呼吸的功能主义界面**。  
目标不是“做得空”，而是让信息层级、留白、材质感与交互反馈都足够克制且可用。

适用场景：品牌官网、作品集、预约页、活动页、生活方式产品、内容页、轻量 SaaS 落地页。  
不优先适用：高密度交易终端、强游戏化产品、依赖高刺激转化的营销页。

---

## 1. 核心原则

- **Less but better**：每个元素都必须承担信息、导航或情绪价值，不做纯装饰堆叠。
- **Ma（間） + Hygge**：留白不是空缺，而是内容的一部分；温暖感来自节奏、材质与比例，而不是颜色堆砌。
- **Wabi-sabi**：允许轻微不对称、自然纹理与手工感，但这种“不完美”必须是可控的。
- **Calm over hype**：优先平静、稳定、可信；弱化刺激性配色、夸张动效和营销感过强的视觉语言。
- **Accessibility first**：Japandi 可以克制，但不能牺牲可读性、可点击性和状态反馈。

---

## 2. 视觉气质边界

### 2.1 应有感受

- 安静
- 温暖
- 自然
- 有秩序
- 不炫技

### 2.2 不应出现

- 霓虹、赛博、玻璃态主导页面
- 过度阴影、发光、强渐变
- 高频动画、弹跳、视差炫技
- 颜色很多但层级不清
- 极简到没有反馈、没有状态、没有可用性

### 2.3 视觉优先级

1. 内容结构
2. 留白节奏
3. 排版气质
4. 低饱和语义色
5. 材质与细节

## 3. Design Tokens

### 3.1 色彩系统（Semantic Color Tokens）

主色策略：以暖中性色为底，深炭色建立信息层级，低饱和自然色只做少量点缀。

| Token                | Hex       | 用途                       | 说明                       |
| -------------------- | --------- | -------------------------- | -------------------------- |
| `bg.canvas`          | `#F5F2ED` | 页面主背景                 | 温暖的非纯白背景           |
| `bg.surface`         | `#EDE6DE` | 卡片、输入区、分组区       | 与背景有轻微层次           |
| `bg.elevated`        | `#F8F5F1` | 浮层、弹层、重点容器       | 比主背景更轻一点           |
| `text.primary`       | `#2D2A25` | 正文、标题、主要数据       | 主文本色                   |
| `text.secondary`     | `#6B665C` | 次级说明、辅助信息         | 不用于核心 CTA             |
| `text.tertiary`      | `#8E877C` | placeholder、弱提示        | 保持可读，不低于可访问阈值 |
| `border.default`     | `#D6CEC4` | 默认边框、分割线           | 主体边界                   |
| `border.strong`      | `#B7AEA2` | 强调边框、选中态           | 不要大面积使用             |
| `accent.primary`     | `#4E463D` | 主按钮、重点文字、关键图标 | 深暖棕灰，替代纯黑         |
| `accent.soft`        | `#8A8175` | hover、标签、弱强调        | 轻度强调                   |
| `accent.nature`      | `#8A9980` | 少量点缀、tag、图标        | 整页占比应极低             |
| `state.success`      | `#6E7F65` | 成功提示                   | 去荧光化绿色               |
| `state.warning`      | `#A07B4E` | 提醒提示                   | 土黄而非高亮黄             |
| `state.error`        | `#9A5A52` | 错误提示                   | 柔和陶土红                 |
| `state.info`         | `#6F7C84` | 中性提示                   | 冷灰蓝，仅用于信息反馈     |
| `state.focus`        | `#7E7468` | 焦点描边                   | 所有交互元素统一使用       |
| `state.disabledBg`   | `#E6DED5` | 禁用背景                   | 与正常态可区分             |
| `state.disabledText` | `#A39A8F` | 禁用文本                   | 不可与正文混淆             |

#### 色彩使用规则

- 背景和表面色占全页视觉面积的 **70–85%**。
- 深色文本和主强调色占 **10–20%**。
- `accent.nature` 控制在 **3–5% 以下**，避免页面发花。
- 禁止大面积使用 `#000000` 和 `#FFFFFF`；需要极致反差时，优先使用 `accent.primary` 与 `bg.elevated`。
- 状态色必须服务于语义，不可只凭“好看”选色。
- 任何状态表达都**不能只依赖颜色**，需同时配合图标、文案或边框变化。

  --color-border-default: #d6cec4;
  --color-border-strong: #b7aea2;
  --color-accent-primary: #4e463d;
  --color-accent-soft: #8a8175;
  --color-accent-nature: #8a9980;
  --color-success: #6e7f65;

  --color-error: #9a5a52;
  --color-info: #6f7c84;

### 3.3 排版系统（Typography）

Japandi 的“温度”更适合体现在标题和局部展示中，正文与 UI 控件仍应优先保证清晰、轻盈与中英文混排稳定。

#### 字体角色

- **Display / Heading**：`'Noto Serif SC', 'Source Serif 4', Georgia, serif`
- **UI / Body**：`Inter, 'Noto Sans SC', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif`
- **Mono / Code**：`'JetBrains Mono', 'IBM Plex Mono', monospace`

#### 使用规则

- 标题可使用 serif，但**正文、表单、导航、按钮优先 sans-serif**。
- 页面内最多使用 **2 个主字体家族 + 1 个等宽字体**。
- 默认字重：`400 / 500`；`600` 仅用于少量标题与关键数据；避免 `700+` 成为常态。
- 中文正文避免过度 letter-spacing；英文大标题可使用 `-0.01em` 到 `0.02em` 微调。

#### 字号与行高

| Token        | Mobile | Desktop | Line Height | 用途               |
| ------------ | ------ | ------- | ----------- | ------------------ |
| `display.xl` | `40px` | `64px`  | `1.08–1.12` | Hero 主标题        |
| `heading.h1` | `32px` | `48px`  | `1.12–1.18` | 页面一级标题       |
| `heading.h2` | `24px` | `36px`  | `1.15–1.22` | 区块标题           |
| `heading.h3` | `20px` | `28px`  | `1.2–1.28`  | 卡片或表单分组标题 |
| `body.lg`    | `18px` | `20px`  | `1.65–1.75` | 大段正文、引导文案 |
| `body.md`    | `16px` | `18px`  | `1.6–1.7`   | 默认正文           |
| `body.sm`    | `14px` | `16px`  | `1.55–1.65` | 辅助信息           |
| `label`      | `13px` | `14px`  | `1.4–1.5`   | 标签、说明、meta   |

#### 行长规则

- 中文正文建议单行 **28–40 个汉字**。
- 英文正文建议单行 **60–72 个字符**。
- 长文阅读区建议宽度 **680–760px**。
- 大标题文字块不要超过 **8–10 个词** 或 **2 行**。

### 3.4 间距、圆角、边框、阴影

#### Spacing Scale

`4, 8, 12, 16, 24, 32, 48, 64, 96, 128`

#### 使用建议

- 组件内边距：`12 / 16 / 24`
- 组件间距：`16 / 24 / 32`
- 区块间距：`64 / 96 / 128`
- 页面上下留白：桌面端至少 `96px` 起

#### Radius Scale

| Token         | 值    | 用途             |
| ------------- | ----- | ---------------- |
| `radius.none` | `0`   | 更克制、更偏日式 |
| `radius.sm`   | `4px` | 按钮、输入框     |
| `radius.md`   | `8px` | 卡片、浮层       |

> 全局圆角建议保持在 `0–8px`，不要同时混用多种大圆角。

#### Border / Shadow

- 默认边框：`1px solid var(--color-border-default)`
- 强调边框：`1px solid var(--color-border-strong)`
- 默认无阴影；如必须分层，仅允许极淡阴影：`0 4px 16px rgba(45, 42, 37, 0.06)`
- 避免多层阴影叠加、彩色阴影、超大模糊阴影

### 3.5 纹理与材质

- 允许使用纸张、亚麻、木纹、石材等自然纹理
- 纹理只作为**辅助层**，推荐透明度 **3–6%**
- **每个主视区最多 1 种纹理**，不要混合多种材料质感
- 有纹理时，必须保证文字对比度与可读性不下降

## 4. 布局与响应式

### 4.1 断点建议

| Breakpoint | 范围          | Grid  | 容器边距 |
| ---------- | ------------- | ----- | -------- |
| `mobile`   | `0–639px`     | 4 列  | `16px`   |
| `tablet`   | `640–1023px`  | 8 列  | `24px`   |
| `desktop`  | `1024–1439px` | 12 列 | `32px`   |
| `wide`     | `1440px+`     | 12 列 | `48px`   |

### 4.2 容器宽度

- 标准内容容器：`720–960px`
- 落地页/品牌页容器：`1080–1240px`
- 长文容器：`680–760px`
- 表单主列：`480–640px`

### 4.3 布局规则

- 以单列或双列为主，避免复杂拼贴式栅格。
- 内容对齐优先左对齐；居中只用于 hero、引言、少量品牌区块。
- 一屏内只保留一个视觉焦点，不要让标题、图片、按钮同时抢主位。
- 大留白必须配合明确的边界和节奏，而不是简单拉大 padding。

### 4.4 Section 间距

- Hero 到下一模块：`96–128px`
- 普通 section 间距：`64–96px`
- 内容组内部：`24–32px`
- 相邻文本元素：`8–16px`

## 5. 核心组件规范

### 5.1 Button

#### 按钮类型

- **Primary**：实底主按钮，使用 `accent.primary`，承载页面主 CTA
- **Secondary**：描边按钮，透明或 `bg.elevated` 底
- **Tertiary / Text**：文字按钮或带轻下划线链接，用于低优先级操作

#### 样式规则

- 高度建议：`40 / 44 / 48px`
- 最小点击区：`44x44px`
- 圆角：`0–4px`
- 边框：`1px`
- 图标尺寸：`16–18px`
- 一屏内主按钮数量建议不超过 `1–2` 个

#### 交互状态

- `default`：安静、明确、对比足够
- `hover`：只做轻微底色/边框/文字变化，不放大、不漂浮
- `active`：亮度略降或内收，避免弹跳缩放
- `focus-visible`：必须有 `2px` 焦点描边或外环
- `disabled`：降低对比并改变 cursor，不可与正常态混淆
- `loading`：保留按钮宽度，显示 spinner 或文案，不要闪烁替换布局

### 5.2 Input / Textarea / Select

- 默认采用浅表面底色 + 1px 边框，或下边框样式
- 输入区高度建议：`44–48px`
- label 必须始终可见，不仅依赖 placeholder
- placeholder 只承担示例，不承担字段说明
- `focus` 必须同时有边框/底色/焦点环变化
- `error` 需要文案 + 图标/边框联动，不能只变红
- `disabled` 需明确降低可交互感

### 5.3 Card / Surface

- 默认使用边框分层，不依赖阴影
- 卡片圆角建议 `0 / 4 / 8px`
- 内边距建议 `16 / 24 / 32px`
- `hover` 只允许轻微背景或边框变化
- 信息卡优先保持纯净，不在一张卡里塞入过多状态色和按钮

### 5.4 Navigation

- 导航优先文字链接，不要做厚重导航条
- 桌面端使用横向轻导航；移动端再切换 hamburger
- 当前页状态建议使用文字加深、底边线或细标记，不用高彩色 pill
- 顶部导航高度建议 `56–72px`

### 5.5 Icon / Divider / Tag

- 图标优先线性风格，`1.5–2px` stroke
- 分隔线应尽量轻，承担节奏作用，而不是强切割
- Tag 使用 `accent.soft` 或 `accent.nature` 的浅层表达，避免鲜艳标签墙

### 5.6 Modal / Drawer / Toast

- Japandi 风格不鼓励频繁打断式弹窗
- modal 仅用于确认、短流程表单、关键阻断信息
- 不允许叠多层 modal
- toast 文案应短、安静、语义清晰，显示后自动消失
- 遮罩建议低对比：`rgba(45, 42, 37, 0.16–0.24)`

### 5.7 Empty / Loading / Error States

- **Empty**：给出原因、下一步动作，不留纯空白
- **Loading**：使用骨架屏、轻线条 spinner 或脉冲块，避免装饰性无限动画
- **Error**：使用柔和错误色 + 清晰文本 + 可恢复操作
- **Success**：避免大面积绿色庆祝式视觉，保持冷静、可信

## 6. 图片、插图与内容呈现

### 6.1 图片风格

- 低饱和、自然光、真实材料感
- 原木、石材、织物、纸张、建筑、留白空间类素材优先
- 尽量避免过强滤镜、霓虹场景、强商业库存感

### 6.2 图片处理

- 可叠加极弱纹理或暖灰滤色，但透明度应低
- 圆角小于界面整体圆角上限
- 不使用夸张投影、描边相框、花哨蒙版

### 6.3 文案气质

- 句子简洁、留有停顿感
- CTA 更适合邀请式语气，而非强促销式语气
- 避免堆砌感叹号、emoji、连续高刺激短语

---

## 7. 动效与微交互

### 7.1 动效原则

- 动效服务于反馈与引导，不服务于“表演”
- 优先使用透明度、颜色、位置的细微变化
- 避免弹跳、果冻、强 easing、复杂 3D 转场

### 7.2 时长建议

| 场景                         | 时长        |
| ---------------------------- | ----------- |
| Hover / Press                | `120–180ms` |
| Panel / Accordion / Dropdown | `180–240ms` |
| Page Transition              | `240–320ms` |

### 7.3 Easing

- 推荐：`ease-out`、`ease-in-out`
- 禁止：`bounce`、`elastic`、大幅 overshoot

### 7.4 Reduced Motion

- 必须尊重 `prefers-reduced-motion`
- Reduced Motion 模式下应关闭位移动画，仅保留必要淡入淡出

---

## 8. 可访问性与可用性

- 正文与关键 UI 的对比度至少满足 **WCAG AA**，普通文本建议 **4.5:1+**
- 正文字号不小于 `16px`
- 所有可点击元素最小点击区不小于 `44x44px`
- 键盘导航必须可达，且 `focus-visible` 清晰可见
- 不移除原生 outline，除非有等价或更强的替代焦点样式
- 表单字段必须有可见 label、错误提示、帮助文本结构
- 不只依靠颜色传达成功、错误、当前选中状态
- 图像必须有合适的 `alt`；纯装饰图需正确标记为装饰性
- 长文本卡片需处理截断、换行和多语言混排问题

---

## 9. 建议的视觉占比

| 元素类型        | 占比建议             |
| --------------- | -------------------- |
| 背景 / 表面色   | `70–85%`             |
| 文字 / 深色结构 | `10–20%`             |
| 主强调色        | `3–8%`               |
| 自然点缀色      | `0–3%`               |
| 状态色          | 按需出现，不作为装饰 |

> 如果页面“看起来不像 Japandi”，通常不是颜色不对，而是强调色、组件密度、纹理数量或动效频率超标。

---

## 10. Never Do（v2）

- 不要把“极简”做成“无反馈、无状态、无层级”
- 不要使用高饱和红、蓝、绿作为大面积主色
- 不要堆叠玻璃态、发光、霓虹、重阴影
- 不要让一页同时出现多种材质纹理
- 不要使用三种以上主字体
- 不要让按钮、卡片、导航各自拥有完全不同的圆角语言
- 不要让 hover、active、focus 三个状态缺失
- 不要用大幅缩放或弹跳来表达交互反馈
- 不要滥用弹窗、抽屉、悬浮提示打断阅读节奏
- 不要在正文中混入大量彩色 emoji 或强装饰图标

## 11. 交付检查清单

- [ ] 是否一眼能看出主次信息与唯一主 CTA
- [ ] 是否存在足够留白，而不是仅靠放大字号撑气质
- [ ] 文本对比度、焦点态、禁用态是否明确
- [ ] 组件是否具备 `default / hover / active / focus / disabled / loading` 的规则
- [ ] 状态色是否低饱和且语义清晰
- [ ] 页面纹理是否控制在低透明度且不过量
- [ ] 中文与英文混排是否稳定、易读
- [ ] 移动端是否保持气质一致，而不是简单缩小桌面稿
- [ ] 是否支持 `prefers-reduced-motion`

## 12. 可直接复用的 AI / 设计提示词

**Prompt 简版：**  
Design a calm Japandi-style web interface with warm off-white backgrounds, deep charcoal typography, minimal shadows, subtle natural textures, generous whitespace, refined serif headings, clean sans-serif body text, low-saturation earth tones, quiet interaction states, and strong accessibility.

**Prompt 中文版：**  
做一个 Japandi 风格网页界面：整体克制、安静、温暖，以暖白和米灰为背景，深炭色作为主文字，少量鼠尾草绿或暖灰棕作点缀；大量留白、弱纹理、轻边框、几乎无阴影；标题可带一点 serif 气质，正文和表单保持清晰现代；组件需要完整 hover、focus、disabled、loading 状态；避免高饱和色、强渐变、玻璃态、重阴影和夸张动画。
